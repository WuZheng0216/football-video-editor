# 使用说明：足球视频剪辑器

## 1. 当前版本定位
这是一个本地运行的桌面版足球视频剪辑与分析原型，当前重点是：
- 导入比赛视频
- 在多轨时间线上做基础剪辑
- 调用本地 Python AI 能力做检测、跟踪和效果预览
- 按当前时间线结果导出视频

当前版本不是完整商用剪辑软件，部分 AI 交互仍处于预览或半闭环状态。本文档会明确区分：`已可用`、`部分可用`、`未接入主流程`。

## 2. 环境要求与启动方式
### 2.1 推荐环境
- Windows 10/11
- Anaconda 或 Miniconda
- Node.js 18.17+
- Python 3.10+
- Electron 通过项目依赖安装

### 2.2 启动步骤
1. 打开终端并激活 Python 环境：
   ```powershell
   conda activate tac
   ```
2. 进入 Electron 项目目录：
   ```powershell
   cd C:\Wuz\TAC\football-video-editor\electron-app
   ```
3. 启动开发模式：
   ```powershell
   npm run dev
   ```
4. 如果没有自动弹出桌面窗口，再执行：
   ```powershell
   npx electron .
   ```

### 2.3 重要说明
- 必须在 Electron 桌面窗口中使用。
- 浏览器里打开的 `http://localhost:3000` 仅用于前端开发预览，不代表完整可用功能。
- 如果视频时长读取异常或导出失败，通常与 `ffprobe` 或 `ffmpeg` 元数据读取有关。

## 3. 界面分区说明
### 3.1 顶部栏
- `导入视频`：打开本地视频文件。
- `AI 工具`：展开右侧 AI 面板。
- `导出`：展开右侧导出面板。
- `专注监看`：切换更大监看区。

### 3.2 左侧主工作区
- 上方是监看区：显示视频和 AI 叠加效果。
- 中间是播放/剪辑控制条：播放、逐帧、分割、删除。
- 下方是多轨时间线：当前固定为三轨。

### 3.3 右侧侧栏
- `检查器`：查看和修改当前选中片段的基本属性。
- `AI 工具`：选择 AI 操作、调参数、查看结果摘要。
- `导出`：设置导出参数并按当前时间线导出成片。

### 3.4 时间线轨道说明
- `主视频轨`：完整原始视频，默认整片常驻。
- `AI 效果轨`：放大镜、球员视角等效果片段。
- `素材 / 覆盖轨`：当前主要承载高光候选片段等覆盖内容。

## 4. 基础操作流程
### 4.1 导入视频
1. 点击顶部 `导入视频`。
2. 选择本地比赛视频。
3. 成功后会创建完整主视频轨，并显示监看区、时间线和右侧侧栏。

### 4.2 播放与查看
- 点击监看区视频画面可以播放/暂停。
- 底部控制按钮支持：`播放/暂停`、`上一帧`、`下一帧`。
- 拖动播放器时间滑条可快速定位。
- 支持音量调整和 0.5x 至 2x 倍速播放。

### 4.3 时间线剪辑
当前已支持：
- 点击时间线任意位置定位播放头
- 选中片段
- 拖拽片段移动
- 左右边缘裁剪片段时长
- 启用/禁用片段
- 删除片段
- 选中片段后在播放头处执行分割

说明：
- 主视频轨默认是完整原始素材，不会因为高光候选而被直接替换。
- 当前版本更偏向基础多轨组织，不是完整 NLE 级别的复杂剪辑系统。

### 4.4 导出成片
1. 打开右侧 `导出` 面板。
2. 选择导出模式：
   - `当前时间线`
   - `选中片段`
3. 设置格式、分辨率、质量、帧率、文件名和输出路径。
4. 点击 `开始导出`。

导出逻辑说明：
- 导出基于 `timelineSnapshot`，原则上是“预览即导出”。
- 主视频轨会作为完整基础素材参与导出。
- AI 效果轨中的效果片段会在对应时间窗内叠加到导出结果中。

## 9. 放大镜与球员视角当前交互说明
### 9.1 放大镜
当前监看区支持以下交互：
- 鼠标移动：在部分模式下可用于预览放大中心
- 左键：固定放大点
- 右键：清除手动点并回到自动/混合模式
- 滚轮：调整半径
- `Shift + 滚轮`：调整放大倍率

### 9.2 球员视角
当前监看区支持以下交互：
- 左键：设置起点
- 拖动：手动调整方向
- 滚轮：调整视野长度
- `Shift + 滚轮`：调整开角
- `Ctrl + 滚轮`：调整暗化强度

说明：当前这些交互更适合做预览验证，关键帧是否完整回写仍需后续加强。

## 10. 当前限制与已知问题
### 10.1 文案与界面
- 当前部分界面文案仍可能出现乱码。
- 某些组件已经重写，但 `AIToolsPanel.tsx`、`VideoPlayer.tsx` 等仍可能残留乱码字符串。

### 10.2 AI 结果组织
- 右侧目标列表当前没有完全接入真实目标数据。
- 自动高光生成后，更多是候选结果和时间线标记，而不是完整剪辑片段流程。

### 10.3 导出依赖
- 若本机 `ffprobe` / `ffmpeg` 元数据读取失败，视频时长会异常，导出也会失败。
- 导出前请先确认视频元数据读取正常，尤其是总时长不是 0。

### 10.4 项目能力
- 当前版本以单视频源为主。
- 尚未完成成熟剪辑软件级别的素材管理、项目工程管理和高级特效编辑。

## 11. 常见故障排查
### 11.1 只打开了浏览器，没有真正弹出桌面窗口
在 `electron-app` 目录执行：
```powershell
npx electron .
```
必须使用 Electron 窗口，不要直接在浏览器页面里操作。

### 11.2 播放器没有反应
检查以下几点：
- 是否已经成功导入本地视频
- 当前是否是在 Electron 窗口中
- 视频路径是否有效
- 编码格式是否能被系统和浏览器内核正常读取

### 11.3 时间线只有很短一段
这通常说明视频元数据读取失败，重点检查：
- `ffprobe` 是否可用
- `ffmpeg` 是否可用
- 右侧或状态栏是否提示元数据解析失败

### 11.4 导出失败
先检查：
- 输出路径是否已选择
- 视频时长是否读取成功
- 主视频轨是否存在完整原始素材
- AI 效果是否引用了不存在的结果视频

如果报错中包含 `ffprobe`，优先修复本机 FFmpeg / ffprobe 环境。

### 11.5 AI 功能运行后没有达到预期
当前版本建议这样理解：
- 检测 / 跟踪：适合做结果预览和后续功能接线验证
- 放大镜 / POV：适合交互原型验证
- 自动高光：适合候选片段生成，不适合当成完整成片逻辑

## 12. 推荐使用路径
如果你现在要稳定体验当前版本，推荐按下面顺序使用：
1. 导入一段本地比赛视频
2. 检查视频总时长是否正确
3. 播放视频并在时间线上做基础裁剪
4. 运行一次 `球员检测` 或 `多目标跟踪`
5. 如需效果预览，再尝试 `放大镜` 或 `球员视角`
6. 最后到右侧 `导出` 面板执行导出

这条路径是当前版本最接近“可稳定演示”的使用方式。

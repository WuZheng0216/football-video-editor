# 使用说明：开发部署与日常使用

这是一份给团队成员使用的完整说明，目标是让零基础同学也能在一台裸机上把项目跑起来。

适用对象：

- 第一次接手项目的开发同学
- 需要在新电脑上部署环境的成员
- 需要验证功能、录制演示、做联调的同学

## 1. 先理解这个项目是什么

这个项目不是一个单纯的网页，而是一套“桌面应用 + Python AI 引擎”的组合：

- `electron-app`
  负责桌面应用界面、播放器、时间线、项目保存、导出控制
- `ai-engine`
  负责球员检测、片段级跟踪、多人高亮、自动高光等 AI 能力
- `video-core`
  负责视频处理辅助逻辑

你真正操作的产品入口是 Electron 桌面应用，不是普通浏览器页面。

## 2. 一台裸机上需要什么

建议统一使用下面这套版本，团队协作最省心：

- 操作系统：Windows 10/11 64 位
- Node.js：20 LTS
- Python：3.10 到 3.12
- Conda：Miniconda 或 Anaconda
- FFmpeg：最新稳定版
- FFprobe：通常和 FFmpeg 一起安装

为什么必须装这些：

- Node.js：Electron 和 React 都要用
- Python：AI 引擎要用
- FFmpeg / FFprobe：读取视频元数据、导出视频、部分 AI 流程都依赖

## 3. 给零基础同学的推荐安装顺序

请严格按这个顺序来，不要跳步：

1. 安装 Node.js 20 LTS
2. 安装 FFmpeg，并确认 `ffmpeg` / `ffprobe` 可以在终端里直接执行
3. 安装 Miniconda 或 Anaconda
4. 拉取仓库代码
5. 创建 Python 环境并安装 AI 依赖
6. 安装 Electron 主程序依赖
7. 安装 React 渲染层依赖
8. 启动项目并验证

## 4. 系统级软件安装

### 4.1 推荐方式：命令行安装

使用“管理员身份”打开 PowerShell，执行：

```powershell
winget install --id OpenJS.NodeJS.LTS -e
winget install --id Gyan.FFmpeg -e
```

安装完成后，关闭当前 PowerShell，重新打开一个新的终端窗口。

检查是否安装成功：

```powershell
node -v
ffmpeg -version
ffprobe -version
```

正常情况下：

- `node -v` 应该显示 `v20.x`
- `ffmpeg -version` 能输出版本信息
- `ffprobe -version` 能输出版本信息

### 4.2 如果你更习惯手动安装

也可以手动下载安装程序：

1. 安装 Node.js 20 LTS
2. 安装 FFmpeg
3. 把 FFmpeg 的 `bin` 目录加入系统 `PATH`
4. 重新打开终端
5. 再次执行：

```powershell
node -v
ffmpeg -version
ffprobe -version
```

只要 `ffprobe` 找不到，项目里的“视频时长读取”“导出视频”“部分 AI 处理”都可能异常。

如果你的路径不同，把下面命令里的路径替换成你自己的。

```powershell
cd C:\Wuz\TAC\football-video-editor\electron-app
npm install
```

### 5.4 安装 React 渲染层依赖

```powershell
cd C:\Wuz\TAC\football-video-editor\electron-app\src\renderer
npm install
```

## 7. 如何启动项目

### 7.1 开发模式启动

```powershell
conda activate tac
cd C:\Wuz\TAC\football-video-editor\electron-app
npm run dev
```

这条命令会做两件事：

- 启动 React 开发服务器
- 等待页面起来后拉起 Electron 桌面应用

### 7.2 如果 Electron 没自动弹出

有时网页服务已经起来，但桌面窗口没起来。此时执行：

```powershell
cd C:\Wuz\TAC\football-video-editor\electron-app
npx electron .
```

### 7.3 重要提醒

你真正要操作的是 Electron 桌面窗口，不是浏览器里单独打开的 `localhost:3000` 页面。

## 11. 最常见的问题与解决办法

### 11.1 视频时长显示成 `00:00.03`

大概率是下面两类问题之一：

- `ffprobe` 不可用
- 视频元数据没正确读取

先检查：

```powershell
ffprobe -version
ffmpeg -version
```

如果命令找不到，先修复 FFmpeg 环境，不要继续查前端。

### 11.2 启动时报 `node:path`

这是典型的 Node 版本过低。请升级到 Node.js 20 LTS。

### 11.3 只有浏览器页面，没有桌面窗口

说明 React 服务起来了，但 Electron 没拉起。执行：

```powershell
cd C:\Wuz\TAC\football-video-editor\electron-app
npx electron .
```

### 11.4 导出失败

优先检查：

```powershell
ffmpeg -version
ffprobe -version
```

导出功能高度依赖这两个命令。

### 11.5 Python 包安装失败

一般有几类原因：

- 没激活 conda 环境
- Python 版本太新或太旧
- 网络或镜像问题
- PyTorch / OpenCV 安装不完整

先确认：

```powershell
conda activate tac
python -V
pip -V
```
